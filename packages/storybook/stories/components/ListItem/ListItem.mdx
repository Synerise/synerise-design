# ListItem

A polymorphic list row component that renders one of five visual variants — default, danger, select, divider, or header — and supports prefixes, suffixes, sub-menus, copy-to-clipboard, hover tooltips, and keyboard navigation.

## Quick overview

- Default import: `ListItem` — the main list row component
- Named exports: `HoverableSuffix`, `ListWrapper`, `ListContextProvider`, `useListContext`, `GroupItem`, `HoverTooltip`
- Five `type` variants: `'default'` (interactive row), `'danger'` (red destructive), `'select'` (selection row), `'divider'` (visual separator), `'header'` (section title)
- Two sizes: `'default'` (32px) and `'large'` (50px) — `description` only renders at large size
- `suffixel` and `prefixel` accept either `ReactNode` or an `AddonRenderer` function `(hovered: boolean) => ReactNode`
- Use `HoverableSuffix` to swap suffix content on hover (e.g. show a warning icon at rest, edit/delete actions on hover)
- Integrates with `@floating-ui/react` for keyboard navigation in dropdowns

## Hoverable suffix with HoverableSuffix

`HoverableSuffix` lets you show different content at rest vs on hover. This is the recommended pattern for list items that show a status indicator by default but reveal action buttons when the user hovers.

Pass it inside `suffixel` using the `AddonRenderer` pattern — the `hovered` boolean is provided automatically by `ListItem`.

```tsx
import ListItem, { HoverableSuffix } from '@synerise/ds-list-item';
import Icon, { WarningFillM, EditM, TrashM } from '@synerise/ds-icon';

<ListItem
  suffixel={(hovered) => (
    <HoverableSuffix
      hovered={hovered}
      defaultContent={<Icon component={<WarningFillM />} />}
      hoverContent={<>
        <Button type="ghost" mode="single-icon">
          <Icon component={<EditM />} />
        </Button>
        <Button type="ghost" mode="single-icon">
          <Icon component={<TrashM />} />
        </Button>
      </>}
    />
  )}
>
  Item with hoverable actions
</ListItem>
```

When only `hoverContent` is provided (no `defaultContent`), the suffix is hidden at rest and fades in on hover — useful for actions that should only appear contextually.

## AddonRenderer pattern

Both `prefixel` and `suffixel` accept a function `(hovered: boolean) => ReactNode` instead of plain JSX. This lets you change the prefix or suffix appearance based on hover state without needing `HoverableSuffix`.

```tsx
<ListItem
  prefixel={(hovered) => (
    <Icon
      component={<StarM />}
      color={hovered ? theme.palette['yellow-600'] : theme.palette['grey-400']}
    />
  )}
>
  Star changes colour on hover
</ListItem>
```

## Prefix and suffix visibility triggers

Set `suffixVisibilityTrigger="hover"` or `prefixVisibilityTrigger="hover"` to hide the entire prefix/suffix slot until the item is hovered. The content fades in with an opacity transition.

This is different from `HoverableSuffix` — the visibility trigger controls whether the **entire slot** is visible, while `HoverableSuffix` swaps **what content** is shown within the slot.

## Copy-to-clipboard

Pass the `copyable` prop as an object to enable click-to-copy. On mousedown or Enter, the item copies the value and briefly shows the `copiedLabel` before reverting. The `onClick` handler fires after a configurable delay.

```tsx
<ListItem
  copyable={{
    copyValue: 'text-to-copy',
    copiedLabel: 'Copied!',
    timeToReset: 1500,        // ms before label reverts (default: 1000)
    delayClickEvent: 700,     // ms before onClick fires (default: 700)
  }}
>
  Click to copy
</ListItem>
```

## Hover tooltip

Use `renderHoverTooltip` to display a popover card when the item is hovered. The popover is positioned via `@synerise/ds-popover` and can be customised with `popoverProps` (placement, offset, flip).

```tsx
<ListItem
  renderHoverTooltip={() => <Card>Details about this item</Card>}
  popoverProps={{ placement: 'right' }}
>
  Hover for details
</ListItem>
```

## ListWrapper with shared click handler

`ListWrapper` provides a shared `onClick` handler to all descendant `ListItem` components via React context. This avoids passing `onClick` to each item individually.

```tsx
import ListItem, { ListWrapper } from '@synerise/ds-list-item';

<ListWrapper onClick={(itemData) => console.log(itemData.key)}>
  <ListItem itemKey="a">Option A</ListItem>
  <ListItem itemKey="b">Option B</ListItem>
  <ListItem type="divider" />
  <ListItem itemKey="c" type="danger">Delete</ListItem>
</ListWrapper>
```

For custom list containers that also need popover delay configuration, use `ListContextProvider` directly.

## Notes and tips

- **`description` is silently ignored at default size.** Only renders when `size="large"`. No warning is shown.
- **`suffixVisibilityTrigger` and `HoverableSuffix` are complementary.** The trigger controls whether the entire suffix slot is visible; `HoverableSuffix` controls what content is shown within the slot. Using `suffixVisibilityTrigger="hover"` with `HoverableSuffix` that has `defaultContent` means the default content is also hidden until hover.
- **`text` prop wins over `children`.** If both are provided, `text` is rendered.
- **`subMenu` items are always in the DOM** — they use `max-height: 0` with `overflow: hidden` for the collapse animation, not conditional rendering.
- **`copyable` as boolean is legacy.** Always prefer the object form `copyable: { copyValue: '...' }` for new code.
- **`itemKey` vs `key`** — use `itemKey` for the stable identifier that appears in `onClick` callbacks. React's `key` should be used only for list rendering.
- **Keyboard navigation** requires a `ListContextProvider` ancestor. ListItem registers with `@floating-ui/react` via `useListItem()` for arrow-key navigation in dropdowns.
