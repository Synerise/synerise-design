# ListWrapper

A container for `ListItem` rows that provides a single shared `onClick` handler to every descendant item, applies list padding/background, and can optionally collapse a long list behind a show-more/less toggle via `maxToShowItems`.

## Quick overview

- Named export: `import { ListWrapper } from '@synerise/ds-list-item'`
- Wrap `ListItem` elements to share one `onClick` handler across all of them (via React context) instead of wiring each item individually
- Applies the standard list container styling (padding, white background, rounded corners); extend it with `styled(ListWrapper)`
- `onClick` reaches items at **any nesting depth** — items may be wrapped in `div`s, fragments, or a virtualised list
- `maxToShowItems` collapses the list to the first N items and renders a show-more/less toggle — but it counts **direct children only** (see below)
- Toggle button labels are localised via react-intl by default (`DS.LIST-ITEM.SHOW-MORE` / `DS.LIST-ITEM.SHOW-LESS`) and can be overridden per-instance through `texts`

## Basic usage

Pass `ListItem`s as children and a single `onClick`. Each item reports its `itemKey` back through the shared handler.

## Limiting visible items with maxToShowItems

Set `maxToShowItems` to collapse the list to the first N items. A ghost-primary toggle button appears only when there are more items than the limit, letting the user expand ("Show more") and collapse ("Show less") the remainder.

## Direct children vs. composed children

There are two distinct behaviours, and they treat nesting differently:

- **Shared `onClick`** works regardless of how the items are nested — it flows through React context, so items can sit inside wrapper `div`s, fragments, or a virtualised list.
- **`maxToShowItems` clipping** counts and slices **direct children only**. Internally the wrapper does `React.Children.toArray(children).slice(0, maxToShowItems)`, so each item you want counted toward the limit must be a **direct child** of `ListWrapper`.

If you wrap your items in an intermediate element, the wrapper sees that element as a single child, and the limit will not behave as intended:

```tsx
// ✅ Counted correctly — each ListItem is a direct child
<ListWrapper maxToShowItems={3} onClick={handleClick}>
  {items.map((props) => (
    <ListItem key={props.itemKey} {...props} />
  ))}
</ListWrapper>

// ⚠️ maxToShowItems counts ONE child (the <div>), not the items inside it
<ListWrapper maxToShowItems={3} onClick={handleClick}>
  <div>
    {items.map((props) => (
      <ListItem key={props.itemKey} {...props} />
    ))}
  </div>
</ListWrapper>
```

Fragments and container components (including virtualised lists) are treated the same way. When you need `maxToShowItems`, keep `ListItem`s as direct children of `ListWrapper`.

## Virtualisation

For very long lists you can compose the children prop with a virtualised renderer (e.g. `react-window`) to only mount the visible rows. Shared `onClick` still works because it propagates through context.

Because the virtualised list is a single composed child, `maxToShowItems` does **not** apply to this pattern — use one or the other.

## Notes and tips

- **`maxToShowItems` requires direct children.** Wrapping items in a container, fragment, or virtualised list makes the wrapper count that single element, not the items inside it — clipping then won't match the visible item count.
- **The toggle only renders when needed.** If the number of direct children is less than or equal to `maxToShowItems`, no toggle button appears.
- **Localised, accessible toggle.** When `maxToShowItems` is used, `ListWrapper` reads its default labels through react-intl, so it must render inside a `DSProvider` (as every Synerise app does). The toggle carries `aria-expanded` reflecting the collapsed/expanded state for assistive tech.
- **`onClick` on both `ListWrapper` and a child.** If a child `ListItem` also passes its own `onClick`, both the wrapper handler and the item handler fire.
- **`popoverDelay` is not forwarded.** `ListWrapper` only forwards `onClick` to the underlying `ListContextProvider`; if you need a custom `HoverTooltip` delay, use `ListContextProvider` directly.
- **Custom styling.** Extend the container with `styled(ListWrapper)` — HTML attributes such as `className` and `style` are spread onto the root container element.
