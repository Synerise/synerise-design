# CodeSnippet

Displays code in three variants — **inline**, **single-line**, and expandable
**multi-line** — with optional syntax highlighting, copy-to-clipboard, and an
animated expand/collapse.

## Quick overview

- Import the default export: `import CodeSnippet, { CodeSnippetType } from '@synerise/ds-code-snippet'`.
- **`children` must be a plain string**, not JSX — all variants type it as `string`.
- Pick the variant with the `type` prop (`CodeSnippetType.INLINE` is the default).
- Syntax highlighting (`colorSyntax`), expand/collapse, `wrap`, `rows`, and
  `overscrollBehavior` apply to the **multi-line** variant only.
- Highlighting loads languages **asynchronously** — expect a brief flash of
  unstyled code before colours apply.

## Inline

The default variant. Renders an inline `<code>` element — no copy button.

## Single-line

A single-line block with a copy-to-clipboard icon and tooltip.

## Multi-line

A scrollable, multi-line block with optional syntax highlighting and an
expand/collapse button that appears once the content exceeds `rows`.

### Initially expanded

Use `expanded` to render the block expanded on first mount. It sets the initial
state only — the component controls expand/collapse itself afterwards.

`hideExpandButton` and `hideCopyButton` remove the respective controls.

## Notes & tips

- `colorSyntax`, `languages`, `rows`, `wrap`, `overscrollBehavior`,
  `hideExpandButton`, and `hideCopyButton` are **multi-line only** — they are
  silently ignored for the inline and single-line variants.
- `expanded` sets the **initial** state only; the component manages
  expand/collapse internally after mount.
- The expand button only appears once the actual line count exceeds
  `rows + 1`; the initial height is derived from `rows`.
- `overscrollBehavior` defaults to `'contain'`, which keeps a scroll gesture
  inside the code block. Set `'auto'` if you want the page to keep scrolling
  once the block reaches its edge.
