# Image

The Image family renders images as thumbnails, groups them into a gallery, and
opens a full-screen preview (lightbox) with zoom, pan and navigation. It is
published as `@synerise/ds-image` and exposes three components plus a helper hook:

- **`Thumbnail`** — a single inline image tile with a fixed aspect ratio and size,
  an optional grey backdrop, `contain`/`cover` fit, an Empty state, a broken-image
  fallback, and an optional delete-on-hover affordance.
- **`Gallery`** — renders many `Thumbnail`s that share the same aspect ratio, size,
  background and object-fit; clicking a thumbnail opens the preview at that image.
- **`ImagePreview`** — a controlled full-screen lightbox: zoom/pan, a bottom-center
  toolbar with navigation (`‹ N of M ›`), zoom controls, and a download link.
- **`useImagePreview`** — owns the preview's open/index state so it can be opened
  from *any* element (a list row, a button, a table cell), not only a gallery.

## Thumbnail

A single tile. `aspectRatio`, `size`, `background` and `objectFit` define the box;
the image fills it according to `objectFit`. Omit `src` for the Empty state, and a
broken `src` swaps in the `fallback`.

## Gallery

A gallery applies its visual props uniformly to every thumbnail (via context) and
owns a single shared preview. Clicking any thumbnail opens the preview at that
image, with navigation across the whole set.

<Canvas of={GalleryDefault} />

## Preview

The preview is a controlled component (`open`, `index`, `onIndexChange`,
`onClose`). Navigation (`‹ N of M ›`) appears only when there is more than one
image; arrow keys page through the set; Escape, the close button and a backdrop
click all close it. Zoom in with the toolbar, the mouse wheel or double-click —
**while zoomed in, drag the image with the mouse to pan** (the cursor turns into
a grab handle); double-click resets. The toolbar's download control saves the
current image. The overlay fades in/out on open/close.

## Triggering the preview from anywhere

Because `ImagePreview` is controlled, any element can open it. `useImagePreview`
removes the boilerplate: it returns `open`/`close`/`next`/`prev` and a
`previewProps` object to spread onto `<ImagePreview>`.

```tsx
const preview = useImagePreview(images);

<ListRow onClick={() => preview.open(2)} />
<ImagePreview {...preview.previewProps} />;
```

## Labels & tooltips

Every icon control (zoom in/out, next/previous, download, close, delete) shows a
tooltip and carries an `aria-label`. The strings come from DS translations
(`react-intl`, ids `DS.IMAGE.*`) and fall back to English defaults — `Zoom in`,
`Zoom out`, `Next image`, `Previous image`, `Download`, `Close`, `Delete`.
Override any subset via the `texts` prop (`Partial<ImageTexts>`) on `Gallery`,
`ImagePreview`, or `Thumbnail`:

```tsx
<ImagePreview {...preview.previewProps} texts={{ close: 'Zamknij', download: 'Pobierz' }} />
```

## Accessibility

- The preview overlay is a `role="dialog"` with `aria-modal`, labelled by the
  current image's `alt`; focus is trapped while open and body scroll is locked.
- Interactive thumbnails are keyboard-operable (`role="button"`, `Tab`,
  Enter/Space) with a visible focus ring.
- Always pass a meaningful `alt` for every image.
