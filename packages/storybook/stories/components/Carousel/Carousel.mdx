# Carousel

A fully native (no Ant Design) carousel: slides with `scrollx` / `fade` transitions,
optional autoplay, built-in dot navigation, and an imperative `goTo` / `next` / `prev`
ref. Built to replace the antd `Carousel` that `@synerise/ds-banner` and
`@synerise/ds-popconfirm` depended on — pure styled-components + theme tokens, emitting
`ds-carousel-*` DOM class hooks for styling and tests.

## Quick overview

- Import the default export: `import Carousel from '@synerise/ds-carousel';`
- **Slides**: one child per slide — `<Carousel><div>…</div><div>…</div></Carousel>`.
- **Transition**: `effect="scrollx"` (default) slides the track; `effect="fade"`
  cross-fades in place (inherently single-slide).
- **Multi-slide**: `slidesToShow` renders several slides at once (`scrollx` only);
  `slidesToScroll` sets how many advance per step.
- **Navigation**: built-in `dots` (hidden for a single slide), or drive it yourself
  via the `ref` (`goTo` / `next` / `prev`) — the surface antd's `CarouselRef` gave.
- **Autoplay**: `autoplay` + `autoplaySpeed` (ms). `infinite` loops at the ends;
  `infinite={false}` clamps to the first / last view.
- **Callbacks**: `beforeChange(from, to)` fires as a transition starts;
  `afterChange(current)` ~500 ms later when it completes.

## Notes & tips

- **`ds-carousel-*` class hooks only** — no antd / slick classes. Target the root
  `.ds-carousel`, then `.ds-carousel-list` → `.ds-carousel-track`, slides
  `.ds-carousel-slide` (+ `.ds-carousel-slide-active` on the visible one(s)), and dots
  `.ds-carousel-dots` → `.ds-carousel-dot` (+ `.ds-carousel-dot-active`).
- **Multi-slide** — the track translates by `-index * (100 / slidesToShow)%` and the
  list clips with `overflow: hidden`; each slide is `100 / slidesToShow %` wide.
- **`afterChange` timing** — fires on a 500 ms timer matching the CSS transition
  (cleared on unmount / re-trigger), so it lags `beforeChange`.
- **Autoplay** — only runs while there is more than one view (`count > slidesToShow`)
  and pauses nothing on hover; drive it manually via the ref if you need finer control.
