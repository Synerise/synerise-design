# ItemsRoll

A panel component that displays a paginated, searchable list of items with optional grouping. ItemsRoll provides a header toolbar (item count, search, change-selection button, custom actions), hover-reveal remove icons on each item, and a footer with show more/less pagination and a clear-all confirmation dialog. The entire component is wrapped in a `@synerise/ds-panel`.

## Quick overview

- Import `ItemsRoll` (default export) from `@synerise/ds-items-roll`
- Pass an array of `ItemRollElement` objects via the `items` prop -- each item requires a unique `id` and optionally a `group` string
- Enable the footer (show more/less + clear all) with `useFooter`
- Enable grouped rendering by providing a `groups` array of strings that correspond to each item's `group` field
- Search is built in -- supply `onSearch`, `onSearchClear`, and `searchValue` for controlled search, or set `hideSearch` to hide it
- `isDisabled` hides the remove icon and clear-all button entirely rather than rendering them as disabled
- All user-facing labels are customisable via the `texts` prop or automatically translated via `react-intl` (`DS.ITEMS-ROLL.*` namespace)

```tsx
import ItemsRoll from '@synerise/ds-items-roll';
import type { ItemRollElement, ItemsRollProps } from '@synerise/ds-items-roll';
```

## When to use

**Use ItemsRoll when:**
- You need to display a user-curated list of selected items (e.g. segments, tags, filters) with the ability to search, paginate, remove, and clear
- Items may belong to logical groups that should be visually separated
- The list can grow large and needs show-more/show-less pagination

**Do not use ItemsRoll when:**
- You need a simple flat list without a toolbar or footer -- use `@synerise/ds-list` instead
- You need a data table with sorting, filtering, and column configuration -- use `@synerise/ds-table` or `@synerise/ds-table-new`
- You need a picker/selection dialog -- ItemsRoll displays already-selected items, not a selection UI

### `ItemRollElement`

Each item in the `items` array. Extends `ListItemProps` (from `@synerise/ds-list-item`) with two additional fields.

    <tr><td>group</td><td>string</td><td>Optional group name; ties the item to a group defined in the groups prop</td></tr>
    <tr><td>...rest</td><td>ListItemProps</td><td>All props from ListItemProps (text, icon, description, suffixel, etc.)</td></tr>

```ts
type ItemRollElement<BaseType extends ListItemProps | MenuItemProps = ListItemProps> =
  BaseType & { id: string; group?: string };
```

## Notes & tips

- **Groups filter items.** When `groups` is provided, only items whose `group` field matches a value in the array are rendered. Items without a matching group are silently hidden -- they do not appear in an "ungrouped" section.
- **`isDisabled` hides, not disables.** Setting `isDisabled` passes `undefined` for `onItemRemove` and `onClearAll` to child components, which fully hides those controls rather than rendering them in a disabled visual state.
- **Footer requires items.** The footer only renders when `useFooter` is `true` AND at least one item is visible. An empty list with `useFooter` will not show a footer.
- **Pagination resets on prop change.** The internal `visibleItemsAmount` state resets to `maxToShowItems` whenever that prop changes, so parent components can control the visible count externally.
- **Search is controlled.** The component does not filter items internally. The parent must handle `onSearch` and pass a filtered `items` array along with the `searchValue`.
- **Clear-all uses Popconfirm.** The clear-all button always shows a confirmation dialog before calling `onClearAll`. The confirmation labels are customisable via `texts`.
- **`renderCount` replaces the default count.** When provided, `renderCount` completely replaces the default "Items: N" label, so you can show ratios like "Items: 5 / 500".
