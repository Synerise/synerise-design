# Modal

A portal-based modal dialog with structured header (title, tabs, actions), a slot-based footer, async handler support, scrollable height capping, and an imperative API. Native React implementation — no Ant Design dependency.

## Quick overview

- Import `Modal` (default) for declarative usage, or `showModal` for imperative usage outside the component tree
- Six predefined sizes: `small` (520px), `medium` (792px), `large` (1044px), `extraLarge` (1280px), `fullSize` (100%), `fullScreen` (100% fixed)
- `onOk` and `onCancel` both support `Promise`-returning handlers — modal stays open until the promise resolves
- `maxViewportHeight` constrains modal height and wraps children in a scrollbar (sheet-style, anchored to bottom)
- `footer={null}` removes the footer; `footer={<Custom />}` overrides it; omit for the default Cancel + Apply buttons
- `ref` handle exposes `scrollToTop()` and `scrollToBottom()` for programmatic scroll control
- Sub-components `ModalTitle`, `ModalFooter`, and `TitleContainer` are exported for advanced composition

## Notes & tips

- **`destroyOnClose={false}` preserves children** — when the modal closes, children stay mounted (just hidden via `display: none` on the root element). This preserves form state, scroll position, and hook state across open/close cycles.
- **Async `onCancel` closes automatically** — if `onCancel` returns a promise, the modal awaits it then calls `closeModal()`. Sync `onCancel` does not auto-close; the parent must set `open={false}`.
- **`showModal` uses PortalRenderer** — it renders via `setPortalContent` from `ds-core`, so a `DSProvider` with `PortalRenderer` must exist in the tree.
- **Close button placement differs in `blank` mode** — absolutely positioned top-right in blank mode; inside the header action buttons row in normal mode.
- **`maxViewportHeight` anchors to bottom** — the modal gets `position: fixed; bottom: 0` (sheet-style), not centered.
