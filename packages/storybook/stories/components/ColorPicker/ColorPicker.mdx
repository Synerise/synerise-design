# ColorPicker

An input-triggered dropdown colour picker built on `react-colorful`, with a hex
input, copy-to-clipboard, and an optional saved-colour swatch section.

## Quick overview

- Import the default export: `import ColorPicker from '@synerise/ds-color-picker';`
- **Semi-controlled** — `value` initialises the internal state on mount; drive it
  with `value` + `onChange` (which fires a normalised 6-digit hex on every valid
  change). Accepts hex (`#fff`, `#ffffff`) or CSS named colours (`red`).
- The picker opens in a `Dropdown` overlay. When `disabled` or `readOnly`, the
  dropdown is **not rendered at all** — only the trigger input shows.
- Set `isShownSavedColors` to reveal the swatch section: a "+" creator button
  plus a grid of `maxSavedColors` slots. Saved colours fill the leading slots;
  the rest render as empty placeholders.
- `size` (`S` / `M` / `L`) controls the spectrum height (136 / 168 / 200 px).

## Saved colours

With `isShownSavedColors`, the swatch section renders below the hex input.
Clicking the "+" button saves the current colour; clicking a swatch applies it
(the selected swatch shows a centred white dot). `maxSavedColors` (default `9`)
caps the number of slots — older entries drop off the front.

<Canvas of={SavedColors} />

## Error state

`errorText` shows a message below the trigger and applies error styling; use
`error` for styling without a message.

## Disabled

When `disabled` (or `readOnly`), the component renders only the trigger input —

## Notes & tips

- **Semi-controlled**: the initial `value` → local-state sync runs only on mount.
  Later external `value` changes are **not** reflected; drive the component
  consistently through `value` + `onChange`, or remount via `key` to reset.
- **`onBlur` reverts to last valid**: typing an invalid colour in the trigger and
  blurring restores the last valid hex.
- **Browser-only helpers**: `standardizeColor` uses `<canvas>` and
  `isValidTextColor` uses `new Option().style` — both throw in SSR/Node.
- **Swatch grid**: the swatch section is built from plain styled elements
  (`.ds-color-swatch` for filled, `.ds-color-swatch-placeholder` for empty), not
  `ds-tags`/`ds-button`. The selected swatch shows a centred white dot; the
  selection highlight clears on any text/hex input change.
