# Autocomplete

A fully native (no Ant Design) autocomplete: an autosizing text-input trigger
plus a floating options dropdown built on `@synerise/ds-dropdown`, wrapped in a
`FormField` label/description/error layer.

## Quick overview

- Import the default export: `import Autocomplete from '@synerise/ds-autocomplete';`
- **The component does not filter** — it renders `options` (or `Autocomplete.Option`
  children) as given. Filter server-side via `onSearch`, or opt into client-side
  matching with `filterOption`.
- **Controlled input**: drive `value` + `onChange`; `onSearch` fires on every
  keystroke; `onSelect` fires when an option is chosen.
- **Open behaviour**: focusing, clicking, or typing in the input **opens** the
  dropdown (never toggles it shut); selecting, `Escape`, or an outside click
  closes it. `open` + `onDropdownVisibleChange` allow controlling visibility.
- Options can be provided declaratively via `<Autocomplete.Option value="…">`
  children (read only when the `options` prop is absent).

## Notes & tips

- **No internal filtering** — render the already-filtered `options`/children from
  your `onSearch` handler, or set `filterOption` for antd-style client-side matching.
- **Open is idempotent** — focus/click/type each only open the panel; a single
  interaction can never open-then-close (this fixed a re-click open/close flicker).
- **`handleInputRef`** exposes a native `{ focus, blur, input }` handle (not the
  antd `RefSelectProps`).
- **`autoResize` + `stretchToFit`** — inside a flex item, give the item
  `min-width: 0; flex-grow: 1` so the input can grow/shrink correctly.
