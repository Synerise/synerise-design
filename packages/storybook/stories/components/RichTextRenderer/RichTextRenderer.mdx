# RichTextRenderer

Renders a `RichTextDocument` (the portable JSON block model produced by
`@synerise/ds-rich-text` with `format="document"`) to plain React — with **no editor
or Tiptap dependency**, so it stays tiny (~1 KB gzip) and safe for read-only surfaces
like feeds, previews, and detail views.

## Quick overview

- Import: `import RichTextRenderer from '@synerise/ds-rich-text-renderer';`
- Feed it the `RichTextDocument` you persisted from the editor:
  `<RichTextRenderer document={doc} />`.
- Supported blocks: paragraphs, headings, bullet/ordered lists (nested), code blocks,
  code snippets, images, blockquotes, horizontal rules, tables, and text alignment.
- Text marks: bold, italic, underline, strikethrough, inline code, links.
- Any block type can be replaced per-consumer via `blockOverrides` — e.g. render
  images through your own lightbox component.
- Unknown block types render as `null` instead of crashing, so documents from newer
  editor versions degrade gracefully.

## Notes & tips

- **Keep the editor out of read paths** — this package intentionally has no Tiptap
  dependency; only pull `@synerise/ds-rich-text` where users actually edit.
- Inside a `blockOverrides` component you can delegate the children back to the
  default rendering with `renderBlock` / `renderInline`.
- The document type (`RichTextDocument`, `Block`) is exported by
  `@synerise/ds-rich-text` — the renderer re-uses those types, so both packages stay
  in sync.
- Links render with <code>target="_blank"</code> semantics from the stored HTML
  attributes; sanitize user-generated URLs at write time.
