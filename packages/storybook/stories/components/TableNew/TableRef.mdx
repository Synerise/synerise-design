# Table Ref (Imperative API)

Both `Table` and `VirtualTable` expose imperative methods via a React ref. This allows you to programmatically control the table without managing state.

## Table

`Table` exposes a `TableRef` with a single method:

```tsx
import { useRef } from 'react';
import { Table, type TableRef } from '@synerise/ds-table-new';

const tableRef = useRef<TableRef>(null);

<Table data={data} columns={columns} tableRef={tableRef} rowKey="id" />
```

## VirtualTable

`VirtualTable` exposes a `VirtualTableRef` with scroll control, layout dimensions, and row highlighting:

```tsx
import { useRef } from 'react';
import { VirtualTable, type VirtualTableRef } from '@synerise/ds-table-new';

const tableRef = useRef<VirtualTableRef>(null);

<VirtualTable data={data} columns={columns} tableRef={tableRef} />
```

## Common patterns

### Scroll to a row and highlight it

For `VirtualTable`, the row DOM element only exists while within the virtualization window. Scroll first, then highlight after the row renders:

```tsx
const goToRow = (id: string, index: number) => {
  tableRef.current?.scrollToIndex(index, { align: 'start' });
  requestAnimationFrame(() => {
    tableRef.current?.highlightRow(id);
  });
};
```

### Restore scroll position

Save and restore the scroll offset when navigating away and back:

```tsx
// Save
const offset = scrollContainerRef.current?.scrollTop ?? 0;

// Restore
tableRef.current?.scrollTo(offset);
```

### Calculate sticky header offset

Use `getDimensions` when you need to account for the table's internal chrome in offset calculations:

```tsx
const { titleHeight, headerHeight } = tableRef.current?.getDimensions() ?? {
  titleHeight: 0,
  headerHeight: 0,
};
const totalHeaderOffset = titleHeight + headerHeight;
```
