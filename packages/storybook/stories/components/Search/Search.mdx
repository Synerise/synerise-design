# Search

`@synerise/ds-search` exports two related components: **`Search`** (the full picker — collapsible input plus a virtualised dropdown with recent items, filterable parameters/facets, and contextual suggestions) and **`SearchInput`** (the standalone collapsible input, used on its own when no dropdown is needed).

## Quick overview

- Default export `Search` is a fully controlled, generic component (`Search<Recent, Parameter, Suggestion>`). The parent must own `value` and `parameterValue` and respond to `onValueChange` / `onParameterValueChange` / `onClear`.
- The dropdown shows three optional sections: **recent**, **parameters**, **suggestions**. Each section is configured through a separate `*DisplayProps` object (`title`, `tooltip`, `rowHeight`, `itemRender`, `visibleRows`, `listProps`).
- When `parameterValue` becomes non-empty, the dropdown switches to **suggestions mode** — recent and parameters are hidden until the parameter is cleared.
- Lists are virtualised with `react-window`; provide `rowHeight` for every section. Cap visible rows with `visibleRows` to enable internal scrolling.
- `SearchInput` is the collapsible input on its own. Toggle behaviour: collapses to an icon when clicked outside (`closeOnClickOutside`), or stays expanded (`alwaysExpanded`). Use `focusTrigger` / `toggleTrigger` boolean flip-flops to focus or toggle imperatively.
- Outside-click collapse is gesture-aware: the outside-click handler still fires on `mousedown`, but the collapse animation is deferred to the matching `mouseup`/`touchend`. Adjacent layout-shifting content stays stable for the duration of the user's click.

```tsx
import Search from '@synerise/ds-search';
import { SearchInput, SearchItems, SearchHeader, SearchButton } from '@synerise/ds-search';
import type { SearchProps, SearchLookupConfig, SearchInputProps } from '@synerise/ds-search';
```

### `SearchLookupConfig<T, U, S>`

Maps each dataset to the key used for text matching and display.

```ts
type SearchLookupConfig<T, U, S> = {
  parameters: keyof U;
  recent: keyof T;
  suggestions: keyof S;
};
```

## Notes & tips

- **Fully controlled.** `Search` does not own `value` or `parameterValue`. Selecting an item fires the change callback — the caller must apply the new value back via props.
- **Suggestions mode.** When `parameterValue` is non-empty, only `suggestions` are rendered; `recent` and `parameters` are hidden until the parameter is cleared.
- **`filterLookupKey` shortcut.** If a clicked parameter has a truthy value at `filterLookupKey`, the parameter is treated as a direct result: the input is populated with the parameter's text, `isResultChosen` flips to `true`, and `onParameterValueChange` fires with the `filterLookupKey` value.
- **Arrow-key navigation.** `focusWithArrowKeys` from `@synerise/ds-utils` moves focus across `.ds-search-item` elements. ArrowUp at the top returns focus to the input.
- **`focusTrigger` / `toggleTrigger` are flip-flops.** They are observed via `usePrevious` — the component reacts to a *change* in value, not the value itself. Toggle the boolean each time you want to trigger the action.
- **Virtualisation requires `rowHeight`.** Every section's display props must specify `rowHeight`. Use `visibleRows` to cap the section height when the list grows.
- **Outside-click collapse is gesture-aware.** Both `SearchInput` and `Search` keep the default `useOnClickOutside` events (`mousedown`/`touchstart`) but defer the collapse animation to the matching `mouseup`/`touchend` via a document-level one-shot listener. This avoids layout shift between mousedown and mouseup while still working in host apps that cancel or stop-propagation on `click` events.
- **`renderInMenu` on `SearchItems` is `@deprecated`.** Default-exported lists wrapped in `ds-menu` are being phased out — set `renderInMenu={false}` and avoid `Menu.Item` inside `itemRender`, use `ListItem` from `ds-list-item` package instead.
