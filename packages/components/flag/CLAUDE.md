# Flag (`@synerise/ds-flag`)

> Renders a country flag SVG at a configurable size, identified by an ISO 3166-1 alpha-2 country code; returns an empty fragment for unknown codes.

## Package structure

```
src/
  Flag.tsx            — main component; normalises code, looks up icon, renders FlagContainer
  Flag.types.ts       — CountryCode union (250+ codes) + FlagProps
  Flag.styles.tsx     — FlagContainer styled div (size/aspect-ratio logic)
  Flag.const.ts       — FLAG_SIZE_RATIO (15/21) and DEFAULT_SIZE (24)
  Flag.utils.ts       — countryCodes: CountryCode[] array (runtime list of all codes)
  index.ts            — public exports
  icons/
    index.ts          — auto-generated: re-exports all SVG components by country code
    <CC>.tsx          — auto-generated SVG component per country (e.g. PL.tsx, US.tsx)
  __specs__/
    Flag.spec.tsx     — Jest: single render test (class ds-flag-PL present)
build/
  svgr.js             — build script: transforms src/svg/*.svg → src/icons/*.tsx + icons/index.ts
  template.js         — SVGR component template used by svgr.js
```

## Public exports

### `Flag` (default export)

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `country` | `CountryCode \| string` | — (required) | Country code; case-insensitive, hyphens stripped before icon lookup |
| `size` | `number` | `24` | Height of the flag in px; width is calculated via `FLAG_SIZE_RATIO` (15/21 ≈ 0.714) |

No `forwardRef`.

### `countryCodes`

`CountryCode[]` — runtime array of all 258 valid country code strings. Useful for populating selects or validating user input.

### `FlagProps`

Type re-export for consumers that need to type props.

## Usage patterns

```tsx
import Flag, { countryCodes } from '@synerise/ds-flag';

// Basic
<Flag country="PL" />

// Custom size
<Flag country="US" size={32} />

// Case-insensitive, hyphenated sub-national codes supported
<Flag country="gb-eng" />  // → looks up icon "GBENG"

// Populate a select with all available codes
countryCodes.map((code) => <option key={code}>{code}</option>)
```

## Styling

`FlagContainer` is a styled `div` in `Flag.styles.tsx`:
- `display: flex; vertical-align: middle`
- Width and height are driven by the `size` prop and `FLAG_SIZE_RATIO` (15 ÷ 21). Height = `size`; width = `size / FLAG_SIZE_RATIO`.
- No theme tokens — sizing is purely numeric.

## Key dependencies

- `@svgr/core` + plugins — build-time only; transforms raw SVG files into React components (not a runtime dependency)

## Implementation notes

- **Icon lookup** — `Flag.tsx` uppercases the `country` prop and removes all hyphens before looking up the icon from `./icons`. E.g. `"gb-eng"` → `"GBENG"`, `"PL"` → `"PL"`. If no matching icon is found the component renders an empty fragment (`<></>`).
- **Auto-generated icons** — `src/icons/*.tsx` and `src/icons/index.ts` are generated by `build/svgr.js`. Never edit them by hand. To add a new flag: place the SVG in `src/svg/`, run the build script, then commit the generated files.
- **SVG IDs are hashed** — SVGO prefixes every `id` attribute with a string hash of the file content to avoid collisions when multiple flags appear on the same page.
- **`data-testid`** — each generated icon component has `data-testid="ds-flag-{CC}"` on the `<svg>` element.
- **CSS class names** — the `FlagContainer` div always receives `ds-flag` and `ds-flag-{CC}` (uppercased) classes, e.g. `ds-flag ds-flag-PL`.
- **`CountryCode` union includes sub-national codes**: `GB-ENG`, `GB-NIR`, `GB-SCT`, `GB-WLS`, `GB-ZET`, `US-CA`.
- **Test runner is Jest** (not Vitest) — `"test": "jest"` in package.json.
